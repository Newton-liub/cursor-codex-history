# 给外行看的安装说明和功能介绍（Cursor Codex 历史保存）

这份说明是写给“几乎不懂代码”的你。

你可以把它理解成一个“聊天记录保险箱”：
- 它会把 Cursor/Codex 的历史记录备份到你自己的目录。
- 你可以随时查看、导出、归档、恢复、删除。
- 它不会去改 Cursor 内部数据库，所以更安全、更稳。

## 1. 这个工具能做什么

你能用它做 7 件事：
- `reindex`：扫描并建立历史索引（第一次必做）
- `list`：列出会话
- `preview`：预览某个会话
- `export`：导出为 Markdown 文档（可发给别人看）
- `archive`：把会话归档（不删除）
- `recover`：把归档会话恢复回来
- `delete --force`：彻底删除外部备份中的会话

## 2. 它不会做什么

- 不会写回 Cursor 内部数据库
- 不保证“恢复成 Cursor 原生 UI 会话”
- 不会删除你的原始 Cursor/Codex 源日志文件

简单说：它是“外部历史管理器”，不是“官方内部数据库修复器”。

## 3. 安装前准备

你需要：
- Linux 或 macOS
- 已安装 Node.js（建议 22+）
- 已安装 Cursor/Codex 并有过聊天记录

先在终端检查：

```bash
node -v
```

如果能显示版本号，就可以继续。

## 4. 一键安装（推荐）

先进入项目目录：

```bash
cd /path/to/cursor-codex-history
```

执行安装：

```bash
bash scripts/install.sh --with-service
```

这条命令会做两件事：
- 把本项目安装为 Codex skill（默认软链接方式）
- 安装并启动后台同步服务（自动持续备份）

如果提示目标已存在，可以用：

```bash
bash scripts/install.sh --with-service --force
```

## 5. 第一次使用（非常重要）

先做一次全量扫描建索引：

```bash
node scripts/history-cli.js reindex
```

然后列出会话：

```bash
node scripts/history-cli.js list --limit 20
```

如果你看到会话列表，说明安装成功。

## 6. 日常最常用命令

1. 看最近会话

```bash
node scripts/history-cli.js list --limit 20
```

2. 预览某条会话

```bash
node scripts/history-cli.js preview --session-id <会话ID>
```

3. 导出会话成文档

```bash
node scripts/history-cli.js export --session-id <会话ID> --output ./exports/记录.md
```

4. 归档会话

```bash
node scripts/history-cli.js archive --session-id <会话ID>
```

5. 恢复归档会话

```bash
node scripts/history-cli.js recover --session-id <会话ID>
```

6. 删除会话（危险操作）

```bash
node scripts/history-cli.js delete --session-id <会话ID> --force
```

## 7. 后台服务怎么查看

查看服务状态：

```bash
systemctl --user status cursor-codex-history-sync.service
```

重启服务：

```bash
systemctl --user restart cursor-codex-history-sync.service
```

停止服务：

```bash
systemctl --user stop cursor-codex-history-sync.service
```

## 8. 数据都保存在哪

默认历史仓库目录：

```text
~/.cursor-codex-history
```

里面常见子目录：
- `raw/` 原始副本
- `normalized/` 规范化会话
- `exports/` 导出文档
- `archive/` 归档区
- `index.sqlite` 索引数据库

## 9. 卸载方法

普通卸载（保留历史数据）：

```bash
bash scripts/uninstall.sh
```

连历史数据也删除：

```bash
bash scripts/uninstall.sh --remove-data
```

## 10. 常见问题

1. 看不到任何会话
- 先执行 `reindex`
- 确认你之前确实在 Cursor/Codex 里有历史记录

2. 导出失败
- `--output` 必须写在当前目录里面（安全限制）

3. 安装时提示 `target already exists`
- 说明你之前已经装过
- 用覆盖安装：

```bash
bash scripts/install.sh --without-service --force
```

4. 安装服务时报 user systemd 不可用
- 先完成无服务安装，不影响你用 `list/preview/export`：

```bash
bash scripts/install.sh --without-service
```

- 后续在正常登录终端（有 user systemd）再启服务：

```bash
bash scripts/install.sh --with-service --force
```

5. 提示只读文件系统或无法写入 `~/.codex`
- 这通常是受限终端/沙箱环境导致
- 换到你自己的普通终端重试，或设置可写目录给 `CODEX_HOME`

6. 删除后想恢复
- `delete --force` 是不可逆的（除非你另有备份）
- 日常建议先用 `archive`，不要直接 `delete`

## 11. 给外行的建议

- 日常使用 `list / preview / export / archive / recover` 就够了
- `delete --force` 请慎用
- 每次升级后，先跑一次 `reindex` 做健康检查
